# 待跟踪问题

本文档用于持续跟踪后续需要处理的问题，排序规则如下：

1. 简单的先做，复杂的后做
2. bug 修复优先于功能需求
3. 复杂度相同，重要程度高的先做
4. 明确的先做，不明确的后做

后续处理方式：按本文档顺序逐项提取、处理、补充结论。

状态约定：

- 未开始：尚未分析或处理
- 处理中：正在分析、实现或验证
- 已完成：已确认完成
- 暂缓：当前不处理，但保留跟踪

### 3. 文件浏览器搜索支持 `*`、`?` 通配符语法
状态：已完成
说明：

- 若搜索词包含通配符，则按通配符匹配
- 若不包含通配符，则按包含匹配
- 若搜索词包含路径分隔符 `/` 或 `\`，则按完整路径匹配

结论：
- Bridge `search_by_filename` 已支持：无通配符→contains匹配；有通配符→glob_matches匹配；有路径分隔符→匹配相对路径或完整路径
- 内容搜索不变，仅文件名搜索支持通配符

## P3 需进一步澄清/设计

### 6. Web 版本能够在模型思考中的时候，先显示"在思考什么"，Android 端是否需要整体对齐
状态：未开始
说明：这可能不只是一个展示问题，而是整体 SSE 实时更新能力的问题。

- 当前移动端主要依赖增量同步，而不是完全依赖 SSE

- 需要进一步明确目标：是只补齐"思考中内容"的体验，还是整体提升未完成消息的实时更新能力

结论：待补充
