# 连接管理状态机

## 状态列表

| 状态 | 携带数据 | 说明 |
|---|---|---|
| Idle | profile? | 未连接，无活跃会话 |
| ProbingNetwork | profile, attempt | 查网络+缓存，决定走哪条路 |
| WaitingForNetwork | profile, attempt | 等网络恢复 |
| ProbingDirect | profile, attempt | TCP 探测直连 Bridge |
| ProbingGateway | profile, attempt | HTTP 探测网关在线 |
| Connecting | profile, route, attempt, fromCache | 建立 SSE 连接 |
| Connected | profile, route, attempt | SSE 已连接 |
| Recovering | profile, attempt | 退避等待，准备重新探测 |
| Failed | profile, reason | 所有路由不可达 |
| NeedsRepair | profile | Bridge 需要配对授权 |

## 路由类型

| Route | 数据 | 说明 |
|---|---|---|
| Direct | address, port | 直连 LAN/Tailscale |
| Gateway | instanceId | 经云网关中继 |

## 事件列表

| 事件 | 数据 | 来源 |
|---|---|---|
| Connect | profile | ConnectionManager.connect() |
| Disconnect | — | ConnectionManager.disconnect() |
| Retry | — | UI 重试按钮 |
| RepairCompleted | profileId | UI 完成配对 |
| NetworkAvailable | — | NetworkChangeMonitor |
| NetworkLost | — | NetworkChangeMonitor |
| AppForegrounded | — | AppForegroundMonitor |
| CacheDirectOk | route.Direct | EffectExecutor.checkNetwork() |
| CacheGatewayOk | route.Gateway | EffectExecutor.checkNetwork() |
| CacheNone | — | EffectExecutor.checkNetwork() |
| NetworkIsNone | — | EffectExecutor.checkNetwork() |
| ProbeOk | route | EffectExecutor 探测成功 |
| ProbeFail | route, reason? | EffectExecutor 探测失败 |
| BridgeNotBridge | profile | 直连地址不是 Bridge |
| BridgeNeedsRepair | profile | Bridge 需要授权 |
| SseConnected | route | SyncSseClient.Connected |

**注意**：`NetworkIsWifi` / `NetworkIsMobile` 仍在 ConnEvent 中定义，但当前无状态使用。

## 副作用列表

| Effect | 数据 | 执行内容 |
|---|---|---|
| CheckNetwork | profile, attempt | 查网络+缓存 → 发 CacheXxx / NetworkIsNone |
| ProbeDirect | — | TCP 探测直连 → 发 ProbeOk/ProbeFail/BridgeNotBridge/BridgeNeedsRepair |
| ProbeGateway | — | HTTP 探测网关 → 发 ProbeOk/ProbeFail |
| StartSse | route | 按 route 启动 SSE 连接 |
| StopSse | — | 断开 SSE |
| StartBackoff | delayMs | 开始退避定时器 |
| StopBackoff | — | 取消退避定时器 |
| RefreshSessions | — | 刷新会话列表 |
| UpdateLastConnectedAt | profileId | 更新 profile.lastConnectedAt |
| StartDirectCheckLoop | — | 每 30s 探测直连是否恢复 |
| StopDirectCheckLoop | — | 停止直连探测循环 |
| RestartDirectCheckLoop | — | 重启直连探测循环 |
| WriteCacheDirect | profileId | DataStore 写 direct 缓存 |
| WriteCacheGateway | profileId | DataStore 写 gateway 缓存 |
| ClearCache | profileId | DataStore 清除缓存 |

## 状态跃迁表

### Idle

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| Connect(profile) | ProbingNetwork | _state = ProbingNetwork(profile) | 开始连接流程 |
| Disconnect | Idle | _state = Idle() | 重复断开，保持 Idle |

### ProbingNetwork

**onEntry**：CheckNetwork(profile, attempt)

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| CacheDirectOk(route) | Connecting | _state = Connecting(profile, route, attempt, fromCache=true) | 缓存有直连记录，直接连 |
| CacheGatewayOk(route) | Connecting | _state = Connecting(profile, route, attempt, fromCache=true) | 缓存有网关记录，直接连 |
| CacheNone | ProbingDirect | _state = ProbingDirect(profile, attempt) | 无缓存，走探测 |
| NetworkIsNone | WaitingForNetwork | _state = WaitingForNetwork(profile, attempt) | 无网络 |
| Disconnect | Idle | _state = Idle(profile) | 用户断开 |

### WaitingForNetwork

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| NetworkAvailable | ProbingNetwork | _state = ProbingNetwork(profile, attempt) | 网络恢复 |
| AppForegrounded | ProbingNetwork | _state = ProbingNetwork(profile, attempt) | 前台恢复 |
| Disconnect | Idle | _state = Idle(profile) | 用户断开 |

### ProbingDirect

**onEntry**：ProbeDirect

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| ProbeOk(route) | Connecting | _state = Connecting(profile, route, attempt) | 直连可达 |
| ProbeFail | ProbingGateway / Failed | instanceId 非空→ProbingGateway，否则→Failed | 直连不通 |
| BridgeNotBridge | ProbingGateway / Failed | 同上 | 地址不是 Bridge |
| BridgeNeedsRepair | NeedsRepair | _state = NeedsRepair(profile) | 需要授权 |
| Disconnect | Idle | effect: StopSse, _state = Idle(profile) | 用户断开 |

### ProbingGateway

**onEntry**：ProbeGateway

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| ProbeOk(route) | Connecting | _state = Connecting(profile, route, attempt) | 网关在线 |
| ProbeFail | Failed | _state = Failed(profile, "All routes failed") | 网关也不通 |
| Disconnect | Idle | effect: StopSse, _state = Idle(profile) | 用户断开 |

### Connecting

**onEntry**：StartSse(route)

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| SseConnected(route) | Connected | _state = Connected(profile, route, attempt) | SSE 连接成功 |
| SseFailed | ProbingGateway / Recovering | **分支逻辑**：fromCache && Direct && instanceId 非空 → ClearCache + ProbingGateway；否则 → Recovering(attempt+1)，fromCache 时 ClearCache | SSE 连接失败 |
| Disconnect | Idle | effect: StopSse, _state = Idle(profile) | 用户断开 |

### Connected

**onEntry**：RefreshSessions, UpdateLastConnectedAt, WriteCache(Direct/Gateway), Gateway 时 StartDirectCheckLoop

**onExit**：StopDirectCheckLoop

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| ProbeOk(route.Direct) | Connecting | guard: 当前是 Gateway 且探测到直连恢复 → _state = Connecting(profile, route, attempt) | 直连恢复切回 |
| SseFailed | Recovering | _state = Recovering(profile, attempt+1) | SSE 断开 |
| SseStreamClosed | Recovering | _state = Recovering(profile, attempt+1) | SSE 流关闭 |
| NetworkLost | Recovering | _state = Recovering(profile, attempt+1) | 网络丢失 |
| Disconnect | Idle | effect: StopSse, _state = Idle(profile) | 用户断开 |

### Recovering

**onEntry**：StartBackoff(backoffMs(attempt))

**onExit**：StopBackoff

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| BackoffExpired | ProbingNetwork | _state = ProbingNetwork(profile, attempt) | 退避到期，重新探测 |
| NetworkAvailable | ProbingNetwork | _state = ProbingNetwork(profile, attempt) | 网络恢复 |
| Disconnect | Idle | effect: StopSse, _state = Idle(profile) | 用户断开 |

### Failed

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| Connect(profile) | ProbingNetwork | _state = ProbingNetwork(profile) | 重新连接 |
| NetworkAvailable | ProbingNetwork | _state = ProbingNetwork(profile) | 网络恢复 |
| AppForegrounded | ProbingNetwork | _state = ProbingNetwork(profile) | 前台恢复 |
| Retry | ProbingNetwork | _state = ProbingNetwork(profile) | 用户重试 |
| Disconnect | Idle | _state = Idle(profile) | 用户断开 |

### NeedsRepair

| 事件 | 目标状态 | onTriggered | 备注 |
|---|---|---|---|
| Connect(profile) | ProbingNetwork | _state = ProbingNetwork(profile) | 重新连接 |
| RepairCompleted(profileId) | ProbingNetwork | guard: profileId 匹配, _state = ProbingNetwork(profile) | 配对完成 |
| Disconnect | Idle | _state = Idle(profile) | 用户断开 |

## 退避算法

```
delay = min(1000ms × 2^min(attempt, 4), 30000ms)
attempt 0 → 1s
attempt 1 → 2s
attempt 2 → 4s
attempt 3 → 8s
attempt 4 → 16s
attempt 5+ → 30s（上限）
```

## 缓存路由逻辑

1. `ProbingNetwork` onEntry 调 `CheckNetwork` → `EffectExecutor.checkNetwork()`
   - 无网络 → `NetworkIsNone` → `WaitingForNetwork`
   - 有网络 + 缓存=direct → `CacheDirectOk` → `Connecting(fromCache=true)`
   - 有网络 + 缓存=gateway → `CacheGatewayOk` → `Connecting(fromCache=true)`（需 instanceId 非空）
   - 有网络 + 无缓存 → `CacheNone` → `ProbingDirect`
2. `Connected` onEntry 写缓存（Direct 写 direct，Gateway 写 gateway）
3. `Connecting` SseFailed 且 fromCache：
   - Direct 失败 + instanceId 非空 → ClearCache + `ProbingGateway`
   - 其他 → ClearCache（如果 fromCache）+ `Recovering`

## AppForegrounded 处理现状

| 状态 | 是否处理 | 行为 |
|---|---|---|
| Idle | ✅（ConnectionManager 层） | 如果 profile != null → connect |
| ProbingNetwork | ❌ | 无 transition |
| WaitingForNetwork | ✅ | → ProbingNetwork |
| ProbingDirect | ❌ | 无 transition |
| ProbingGateway | ❌ | 无 transition |
| Connecting | ❌ | 无 transition |
| Connected | ❌ | 无 transition | ← **问题所在** |
| Recovering | ❌ | 无 transition | ← **问题所在** |
| Failed | ✅ | → ProbingNetwork |
| NeedsRepair | ❌ | 无 transition |

**核心问题**：app 从后台恢复时，如果 SSE 已断开（或即将断开），`Connected` 和 `Recovering` 状态不处理 `AppForegrounded`，导致：
- `Connected` 状态：SSE 可能已经死了但 OkHttp 超时还没触发，`AppForegrounded` 被忽略，然后等 OkHttp 超时后才收到 `SseFailed` → `Recovering` → 退避
- `Recovering` 状态：退避期间 `AppForegrounded` 被忽略，用户必须等退避到期才能重连
