# 会话模型错误显式展示记录

## 背景观察

用户回忆的关键现场信息：

1. 当时手机与 Web 都出现“消息发送后无人处理”的现象
2. 打开 Web 页面后，可以看到模型错误状态
3. 该错误状态提示需要等待很长时间后才能重试
4. 后来执行中止后，会话恢复正常

## 当前结论

这至少说明两点：

1. 同一会话在某一时刻只会有一个正在推进的处理流
2. 当模型侧出现错误/限流/等待重试状态时，如果移动端不显式展示该状态，用户会误以为系统只是“卡住”，从而无限等待

## 产品含义

Android 端需要像 Web 一样，把会话当前的模型错误/等待重试状态显示出来，而不是只显示一个持续忙碌的等待状态。

## 暂定后续需求

本次“消息窗口增量更新与按需加载”完成后，下一项工作单独处理：

3. 在会话详情页显式展示模型错误、重试等待或类似状态
