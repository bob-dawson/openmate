# 消息自动跟随滚动设计

## 设计目标

将自动跟随逻辑从 Composable 中解耦，实现一个 **UI 无关的纯状态机类** `AutoFollowTracker`：
- 由外部驱动事件输入（滚动、键盘、消息变化、定位等）
- 输出 `shouldFollow`（是否跟随底部）和 `shouldScrollToBottom`（是否需要执行滚动）
- 通过单元测试覆盖所有场景后再接入 UI

## 核心概念

**followBottom**：布尔状态，表示"是否应该自动跟随底部"。
- `true`：新消息/内容更新时自动滚动到底部
- `false`：保持当前位置，不自动滚动

## AutoFollowTracker 类设计

### 内部状态

| 状态 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| `followBottom` | Boolean | `true` | 是否跟随底部 |
| `prevMessageCount` | Int | `0` | 上一次消息数 |
| `isAutoScrolling` | Boolean | `false` | 程序触发的滚动动画中 |
| `isKeyboardAnimating` | Boolean | `false` | 键盘动画中 |
| `isUserNavigating` | Boolean | `false` | 消息定位中 |
| `pendingScrollRequest` | Boolean | `false` | 脉冲信号：是否需要滚动到底部 |

### 事件方法

| 方法 | 说明 |
|------|------|
| `onMessagesChanged(count: Int, isLoading: Boolean)` | 消息数变化。初始加载(0→N)总是触发滚动；后续变化仅 followBottom=true 且非 loading/navigating 时触发 |
| `onContentUpdated()` | 消息内容更新（streaming delta、tool 状态变化等），可能导致布局变化但 count 不变。仅 followBottom=true 且非 navigating 时触发滚动，不改变任何状态 |
| `onScrollStarted(canScrollForward: Boolean)` | 滚动开始。自动滚动/键盘动画/定位期间忽略；canScrollForward=true → 用户上滚 → followBottom=false |
| `onScrollStopped(canScrollForward: Boolean)` | 滚动停止。自动滚动/键盘动画/定位期间忽略；canScrollForward=false → 用户滚到底 → followBottom=true |
| `onKeyboardAnimationStarted()` | 键盘动画开始，标记 isKeyboardAnimating=true |
| `onKeyboardAnimationEnded()` | 键盘动画结束，标记 isKeyboardAnimating=false；如果 followBottom=true 则触发滚动到底部 |
| `onNavigateToMessage()` | 消息定位开始，followBottom=false, isUserNavigating=true |
| `onNavigateComplete()` | 消息定位结束，isUserNavigating=false，**followBottom 保持 false** |
| `onRequestFollow()` | 外部请求跟随（如"滚到底部"按钮），followBottom=true，触发滚动 |
| `onAutoScrollStarted()` | 程序滚动开始，isAutoScrolling=true |
| `onAutoScrollEnded()` | 程序滚动结束，isAutoScrolling=false |

### 输出

- **`shouldFollow: Boolean`** — 当前是否应该跟随底部（UI 据此显示/隐藏"滚到底"按钮）
- **`consumeShouldScrollToBottom(): Boolean`** — 消费脉冲信号，返回是否需要执行一次滚动到底部，调用后自动重置为 false

### 状态转换规则

```
1. onMessagesChanged(count, isLoading):
   - count == 0 → prevMessageCount = 0, return
   - prevMessageCount == 0 (初始加载):
     - isLoading → prevMessageCount = count, return (不滚，等 loading 结束后 UI 调 onContentUpdated 触发)
     - !isLoading → prevMessageCount = count, requestScroll
   - else (非初始加载) → prevMessageCount = count
     - followBottom && !isLoading && !isUserNavigating → requestScroll

2. onScrollStarted(canScrollForward):
   - isAutoScrolling || isKeyboardAnimating || isUserNavigating → ignore
   - canScrollForward → followBottom = false

3. onScrollStopped(canScrollForward):
   - isAutoScrolling || isKeyboardAnimating || isUserNavigating → ignore
   - !canScrollForward → followBottom = true

4. onKeyboardAnimationStarted():
   - isKeyboardAnimating = true

5. onKeyboardAnimationEnded():
   - isKeyboardAnimating = false
   - if followBottom → requestScroll

6. onNavigateToMessage():
   - followBottom = false
   - isUserNavigating = true

7. onNavigateComplete():
   - isUserNavigating = false
   // followBottom stays false — 用户需手动滚回底部或点按钮

8. onRequestFollow():
   - followBottom = true
   - requestScroll

9. onAutoScrollStarted():
   - isAutoScrolling = true

10. onAutoScrollEnded():
    - isAutoScrolling = false

11. onContentUpdated():
    - followBottom && !isUserNavigating → requestScroll
    // 不改变任何状态
```

其中 `requestScroll` = `pendingScrollRequest = true`

### 关键设计决策

1. **定位期间完全忽略滚动事件** — `isUserNavigating=true` 时，`onScrollStarted/Stopped` 全部忽略，避免定位动画经过底部时误设 followBottom=true
2. **定位结束后 followBottom 保持 false** — 不会自动恢复跟随，用户必须手动滚到底或点按钮
3. **键盘动画期间忽略滚动事件** — 防止键盘弹出/收起触发的布局滚动改变 followBottom
4. **键盘动画结束后补滚** — 如果 followBottom=true，补一次滚动到底部
5. **内容更新不改变状态** — `onContentUpdated()` 只在满足条件时触发滚动，不修改 followBottom 等状态
6. **自动滚动期间忽略滚动事件** — 防止程序触发的滚动被误判为用户操作
7. **consumeShouldScrollToBottom 脉冲消费** — 一次性信号，读取后自动重置，避免重复触发

## UI 集成方式

UI 层（Composable）负责：
1. 将 Compose 滚动事件翻译为 `onScrollStarted/Stopped` 调用
2. 将 IME insets 变化翻译为 `onKeyboardAnimationStarted/Ended` 调用
3. 将 Flow 发射的消息列表变化翻译为 `onMessagesChanged/ContentUpdated` 调用
4. 搜索定位翻译为 `onNavigateToMessage/Complete` 调用
5. 读取 `shouldFollow` 决定是否显示"滚到底"按钮
6. 读取 `consumeShouldScrollToBottom()` 决定是否执行滚动
7. 执行滚动前后调用 `onAutoScrollStarted/Ended`
