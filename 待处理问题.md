# OpenMate 待处理问题

## Phase 2

### 文件浏览器只能访问服务器工作目录

**现状**：opencode 的 `GET /file?path=` API 有安全限制（`Instance.containsPath`），只允许浏览服务器工作目录（`ctx.directory`）下的文件。传入绝对路径无效，只能用相对路径。

**影响**：当会话的 `directory`（如 `D:\openmate\android`）与 opencode server 的 CWD（如 `C:\Users\bob_d`）不同时，FilePicker 无法浏览会话工作目录，只能浏览服务器 CWD。

**Phase 2 方案**：自己实现文件浏览器，不依赖 opencode 的文件 API。Android 端直接通过 SSH/SCP 或自定义 Bridge Agent 协议访问远程文件系统，或让 opencode server 支持按 session directory 浏览文件。
