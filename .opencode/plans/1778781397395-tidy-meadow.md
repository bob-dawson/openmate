# 动态 Agent 选择器

## 目标

替换当前硬编码的 build/plan 切换，改为从 `GET /agent` API 动态获取可用 agent 列表，用类似 ModelPickerSheet 的底部弹出界面让用户选择。

## 当前状态

- agent 切换在顶部 MoreVert 菜单里（build/plan 二选一），硬编码
- 底部状态栏有 agent badge（仅当 agent != "build" 时显示），不可点击
- ViewModel `_selectedAgent` 默认 "build"

## API 信息

- `GET /agent` → `List<AgentDto>`，63ms，65KB
- AgentDto 字段：name, description, mode("primary"/"subagent"), hidden, ...
- 需要 SharedPreferences 缓存（弱网场景）

## UI 设计

- agent badge **常驻显示** agent 名称（build 时也显示 "BUILD"）
- badge **可点击**，点击弹出 AgentPickerSheet
- AgentPickerSheet 分两区：Primary（正常样式）+ Subagent（降低视觉重要性，如灰色文字/小号字体）
- 删除 MoreVert 菜单里的 "Mode" 菜单项
- subagent 和 primary 一样可以直接选择使用

## 改动步骤

### 步骤 1: AgentDto + OpencodeApiClient.getAgents()

**新建** `core/network/src/main/java/com/openmate/core/network/dto/AgentDto.kt`

```kotlin
@Serializable
data class AgentDto(
    val name: String,
    val description: String? = null,
    val mode: String = "primary",
    val hidden: Boolean = false,
)
```

**修改** `core/network/.../OpencodeApiClient.kt`

- 加 `suspend fun getAgents(): List<AgentDto>`

### 步骤 2: SessionDetailViewModel — agents 状态 + 缓存

**修改** `feature/session/.../SessionDetailViewModel.kt`

- `_agents: MutableStateFlow<List<AgentDto>>` + `agents: StateFlow`
- `loadCachedAgents()` — 从 SharedPreferences 读（key: `agent_cache::$profileId`）
- `saveCachedAgents()` — 写 SharedPreferences
- `loadAgents()` — 先缓存，再异步刷新 API
- `loadSession` 开头同步调 `loadCachedAgents()`（和 providers 一样预热）
- 默认 agent：新会话取第一个 primary+非hidden，已有会话按当前规则恢复

### 步骤 3: AgentPickerSheet composable

**新建** `feature/session/.../component/AgentPickerSheet.kt`

- ModalBottomSheet，结构类似 ModelPickerSheet
- 无搜索栏（agent 数量少）
- 分两区：
  - Primary agents：正常样式，agent name 加粗 + description
  - Subagents：降低重要性（description 用小号灰色字）
- 选中项高亮
- onDismiss / onSelect 回调

### 步骤 4: SessionDetailScreen UI 调整

**修改** `feature/session/.../SessionDetailScreen.kt`

1. **删除** MoreVert 菜单 "Mode" 项（~L463-470）
2. **修改** agent badge（~L705-718）：
   - 去掉 `if (selectedAgent != "build")` 条件 → 始终显示
   - 加 clickable → `showAgentPicker = true`
   - 样式参考 model chip 的 `Surface`
3. **新增** `showAgentPicker` mutableStateOf + AgentPickerSheet 渲染块

### 步骤 5: 测试更新

**修改** `SessionDetailViewModelTest.kt` — 补 agents 相关 fake/断言

## 验证

1. 编译通过
2. 打开会话 → agent badge 常驻显示 agent 名称
3. 点击 agent badge → 弹出 AgentPickerSheet，分 Primary/Subagent 两区

5. 新会话默认第一个 primary agent
6. 断网时仍可打开选择器（缓存生效）
