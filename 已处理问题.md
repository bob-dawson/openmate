# 已处理问题

从「待跟踪问题」中确认完成的项目归档于此。

## 1. 模型调用出错无提示
结论：已实现 assistant 终态错误卡片展示 + `session.status` 的 retry 细节解析与会话页显式展示，改为 SSE 实时更新 + 轮询兜底。

## 2. 工具调用被中止/取消之后，还会显示执行中转动图标
结论：已补 `EventReplayer` 中 `step.failed` 时把未结束工具统一转成 `error`（`Tool execution aborted`），中止后 spinner 不再残留。

## 3. compact 消息不会展示，导致用户不知道进展
结论：已为顶层 `type="compaction"` 增加可见渲染；已补未结束 compaction 消息的 `runningAnchors` 恢复，计时不再反复从 0 开始。

## 4. 未结束的 Agent 消息计时在退出会话后重新进入会从 0 开始
结论：已按未结束消息的真实年龄恢复 anchor；`SubtaskDetailScreen` 补传 `runningAnchors`。

## 6. 子任务执行中无法导航
结论：已修复 `EventReplayer` 处理 `message.part.updated` tool 分支（去掉了 `messageId != cachedId` 检查，改为通过 callID 查找）；新增 `session.next.tool.progress` 事件处理。真机验证通过。

## 7. question 回复之后的工具展示不直观
结论：`QuestionCard` 与 `QuestionDialog` 已支持自定义答案输入；question 工具完成后展示结构化答案卡片；dismissed question 显示弱提示。

## 8. bash 工具调用获取命令行参数后即可展开，展示名称改为 shell
结论：已将展示名称改为 shell；运行中 bash 只要拿到 command 就提前走 block 展开渲染。

## 9. edit、applypatch、write 支持点击查看编辑的文件
结论：已补 `BlockToolLine` 可点击文件路径；已补 `EventReplayer` 处理 `message.part.updated` patch 分支；已补结果文本兜底解析。真机验证通过。

## 10. 文件浏览搜索结果无排序功能
结论：已补 `sortSearchResults()` 并在搜索结果区域渲染排序入口。真机验证通过。

## 11. 文件浏览入口从菜单提升到标题栏按钮
结论：已在标题栏搜索按钮旁新增 FolderOpen 图标按钮，已从下拉菜单移除重复入口。

## 12. 文件浏览增加关闭按钮 + Back 优先内部导航
结论：已新增 Close 按钮；已添加两个 `BackHandler` 实现目录内回退优先。真机验证通过。

## 13. 消息搜索定位空搜索时三个 tab 统一显示全部
结论：空搜索时三个 tab 统一返回各自角色过滤后的全部消息。

## 14. 用户发送的信息显示"我"而非"你"
结论：已将 label_you 字符串从"你"改为"我"。

## 15. 消息对话框右侧按钮状态对齐 Web
结论：空闲+空文本→灰色发送；忙碌+空文本→红色停止；文本非空→可用发送。

## 16. 模型不支持 variant 选择
结论：已补 `VariantPickerSheet`；模型标签可点击重选；variant 按实例隔离缓存。真机验证通过。

## 17a. 会话 TODO 列表折叠态规则错误
结论：折叠态只从 `in_progress` 与 `pending` 中取前 3 项，不再回填已完成项。

## 18. 未处理的用户消息在 Web 端会固定停留底部
结论：已补 displayMessages，忙碌时排队消息移到列表末尾。

## 19. 打开会话时有时不会自动滚动到最后一条消息
结论：已增加初始加载延迟到 300ms，使用立即 scrollToItem 代替仅依赖动画滚动。

## 21. 工具消息中的 question 回复展示需要更直观
结论：已在第 7 条中统一处理。

## 22. 消息定位针对"我的消息"支持长按菜单
结论：已添加长按上下文菜单支持复制；"回滚到"待 opencode 提供 API。
